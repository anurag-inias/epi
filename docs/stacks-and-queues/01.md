# Implement a stack with max API

## Description

Design a stack that includes a max operation, in addition to push and pop. The max
method should return the maximum value stored in the stack.

## Stub

```java
public static class Stack {

  public boolean empty() {
    return true;
  }

  public Integer max() {
    return 0;
  }

  public Integer pop() {
    return 0;
  }

  public void push(Integer x) {
  }
}
```

## Solution

??? "Approach 1"

    We can create a stack that saves not just the number, but the max number seen so far.

    ```java
    public static class Stack {

      private static class Pair {
        int value;
        int max;

        Pair(int value, int max) {
          this.value = value;
          this.max = max;
        }
      }

      private final List<Pair> items = new ArrayList<>();

      public boolean empty() {
        return items.isEmpty();
      }

      public Integer max() {
        return items.getLast().max;
      }

      public Integer pop() {
        return items.removeLast().value;
      }

      public void push(Integer x) {
        int max = x;
        if (!items.isEmpty()) max = Math.max(max, items.getLast().max);

        items.add(new Pair(x, max));
      }
    }
    ```

??? "Approach 2"

    We use a regular stack + an integer `max` to keep track of the maximum element seen so far.

    $\text{push}(x)$
    : - If $x \le \text{max}$ then push the element as is.
      - otherwise, $\text{push}(2x - max)$ and update $\text{max} = x$.

    Let's assume our stack in the beginning looks like this, with the largest element $m$ at the top:

    $$
    \begin{align}
    \text{stack} &= [\dots, m] \\ 
    \text{max} &= m
    \end{align}
    $$

    === "handling value $k \le m$"

        if we push some value smaller or equal to the $max$ seen so far, we get:

        $$
        \begin{align}
        \text{stack} &= [\dots, m, k] \\ 
        \text{max} &= m
        \end{align}
        $$

        popping this value is straightforward

        - $max$ remains unchanged.
        - $\text{pop}() = k$ same as a typical stack.

    === "handling value $M > m$"

        we push element $M > m$ into the stack, by transforming it so:

        $$
        \begin{align}
        \text{stack} &= [\dots, m, 2M - m] \\ 
        \text{max} &= M
        \end{align}
        $$

        Note that $\text{top} > \text{max}$ which we can use to distinguish this case from the previous one.

        $$
        \begin{align}
        M &> m \\
        2M &> m+M \\
        2M - m &> M \\
        \text{top} &> \text{max} \\
        \end{align}
        $$

        to pop a value from the stack

        - update $\text{max} = 2 \cdot \text{max} - \text{top} = 2M - (2M-m) = m$.
        - $\text{pop}() = max$ (prior to updating $max$).

    The full implementation is as follows:

    ```java
    public static class Stack {

      private int max;
      private final List<Integer> items = new ArrayList<>();

      public boolean empty() {
        return items.isEmpty();
      }

      public Integer max() {
        return max;
      }

      public Integer pop() {
        int top = items.removeLast();
        if (top <= max) {
          return top;
        } else {
          int backup = max;
          max = 2 * max - top;
          return backup;
        }
      }

      public void push(Integer x) {
        if (items.isEmpty()) {
          items.add(x);
          max = x;
        } else if (x <= max) {
          items.add(x);
        } else {
          items.add(2 * x - max);
          max = x;
        }
      }
    }
    ```
